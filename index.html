Документация по плагину Skript

Skript - это плагин для Bukkit, популярного серверного мода Minecraft. Это позволяет администраторам сервера легко модифицировать работу Minecraft, ничего не программируя. Это достигается с помощью триггеров, каждый из которых представляет собой набор условий и эффектов. Каждый раз, когда вызывается триггер, все условия проверяются, и, если все выполняются, эффекты выполняются. Основная идея Skript - это настраиваемость: Каждый может сделать свой сервер уникальным даже без пользовательских плагинов Гораздо быстрее изменить некоторые сценарии самостоятельно, чем попросить разработчиков плагинов написать некоторые функции, которые вы хотите Также нет необходимости иметь десятки маленьких плагинов, если вы можете сделать то же самое с этим одним плагином Вы можете скачать последнюю стабильную версию Skript здесь (см. Страницу BukkitDev для журналов изменений и дополнительной информации). Как уже упоминалось, Skript работает с триггерами, которые добавляют функциональность этому плагину. Триггеры могут быть очень простыми и состоять только из некоторых условий и эффектов, таких как это простое исправление: 
on break of glass:
    drop glass
Скрипт также можно использовать для создания пользовательских команд. По сути, это триггеры, которые выполняются после ввода определенной команды. Далее определяется простая команда, которая позволяет получить идентификатор элемента, держа его в руке и введя / id: 
command /id:
    description: Find the ID of the item you're holding
    trigger:
        message "You're holding a %type of tool% whose ID is %id of tool%."
Скрипт также поставляется с опцией непосредственного выполнения эффектов из чата, интерпретируя сообщения чата, которые начинаются с настраиваемого символа ('!') По умолчанию, как эффекты. Это по существу позволяет использовать эффекты в качестве «команд», например ! ремонт инструмента сделает то, что он подразумевает. Поскольку эти команды являются очень мощными (и, таким образом, их можно злоупотреблять), только игроки с разрешениями skript.effect могут использовать их, если они вообще включены (это не по умолчанию для OP!). Синтаксис скрипта не ограничивается такими простыми примерами, как выше, но имеет несколько других функций, таких как циклы, условные выражения и т. Д. Ниже приведен пример более продвинутого триггера, который заправляет печи, используя топливо из окружающих сундуков: 
on burn of fuel:
    fuel slot of the block is empty
    loop blocks in radius 1:
        loop-block is chest
        loop items of type fuel:
            loop-block contains loop-item
            remove loop-item from loop-block
            set fuel of the event-block to loop-item
            stop trigger
